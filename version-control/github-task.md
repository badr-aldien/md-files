# Assignment: Version Control Day 2

## Objective

Every team will collaborate to create a single README file hosted on GitHub. You will practice the **Feature Branch Workflow**, resolve **Merge Conflicts**, and use the **Command Line Interface (CLI)** for all Git operations.

## The Rules

1.  **CLI Only**
    All Git actions (cloning, branching, adding, committing, merging, and pushing) **must** be done via the terminal.

2.  **No Direct Pushing to Main branch**
    No one is allowed to push code directly to the main branch (except for the initial setup). All changes must come through **Pull Requests (PRs)**.

3.  **Conflict Resolution**
    You must resolve any conflicts **locally** on your machine using the CLI—do not use the GitHub "Web Editor" to fix conflicts.

---

## Phases

### Phase 1: Repository Setup (Leader Only)

- **Repo**: Create a new public repository on GitHub and add teammates as Collaborators.
- **Init**: Clone the repository, create the README file with a team manifesto title, and push to the main branch.

### Phase 2: Feature Branches (Everyone)

- **Branch**: Clone the repository and create a new feature branch using your name.
- **Edit**: Update the title to a team name and add a personal bio section.
- **Upload**: Stage your changes, commit them, and push your branch to GitHub.

### Phase 3: Pull Requests & Merging

- **PR**: Open a Pull Request into the main branch on GitHub.
- **Merge**: The first PR to be reviewed can be merged immediately; subsequent PRs will likely trigger conflicts.

### Phase 4: Conflict Resolution

- **Sync**: Pull the latest changes from the main branch, then merge them into your feature branch.
- **Resolve**: Manually fix conflict markers in the README file, then stage, commit, and push the resolution.
- **Finalize**: Complete the merge on GitHub.
